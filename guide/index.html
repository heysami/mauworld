Daily picture

Use these first when you want rhythm, motion, and triage.

01

Today

Your morning board.

Start here to see active tasks, what is scheduled today, live blockers, and the freshest saved memory in one pass.

  • Open when you want a quick operational pulse without drilling into detail.
  • Jump directly into in-progress tasks from the page.
  • Catch approvals, failed runs, and new memory before they drift out of view.
02

Wallet

See spend, call time, audio, and characters in one time window.

Wallet is for cost and usage shape. It also carries receipt-driven personal spending charts that can be bucketed by day, week, month, or year.

  • Use it when you need to explain where money or usage moved.
  • Split by merchant or category when the chart needs more texture.
  • Best paired with the finance sync routine when receipts are already flowing.
03

MauOffice

The live pixel office.

MauOffice turns activity into a living scene. It is the page to open when you want a social pulse instead of a spreadsheet feeling.

  • Focus rooms when you want to narrow the scene down.
  • Open actor chat when you need to talk to a specific live presence.
  • Useful for staying oriented without reading every raw event.
04

Tasks

The operational kanban.

Tasks turns sessions and delegated runs into a live board. Root work, blockers, review loops, and project-tagged slices all stay visible.

  • Filter by project when one build thread needs the whole frame.
  • Use the done window when recent completions are drowning the board.
  • Open previews from here when a task already spawned an interactive artifact.
05

Workshop

Preview what the system is building.

Workshop keeps recent artifacts, saved outputs, and agent-app ideas tied back to the task and project that produced them.

  • Use Recent when you want the newest build outputs first.
  • Use Saved when you need the shortlist that already matters.
  • Use Agent Apps when life-improvement ideas are turning into real helpers.
06

Calendar

The time view for routines, cron, approvals, and activity.

Calendar is where operational timing stops feeling abstract. Move between month, week, and day, then filter by cron, user activity, group activity, or approvals.

  • Use month view to see load shape at a glance.
  • Use week and day views when overlap, timing, or congestion matters.
  • Best when you need to answer why something felt busy.
07

Routines

The people-facing layer over automation.

Routines projects recurring behavior into a calmer, human-readable view: schedules, next runs, last runs, preview windows, and whether a routine is paused.

  • Open this before changing automation so you understand the shape first.
  • Use it to confirm that cadence still matches the life it is serving.
  • Especially useful for daily check-ins and finance sync loops.

Planning and growth

These pages hold longer arcs, slower context, and cleaner handoffs.

08

Business

Private venture dossiers without a forced questionnaire.

Business keeps active ventures, profiling gaps, open questions, and the next research or brainstorming entry points in one owner-private place.

  • Use it when an idea is still becoming legible.
  • Gaps do not block progress; they just show where clarity is still thin.
  • Projects stay linked underneath instead of floating loose.
09

Projects

The handoff board from idea to build.

Projects shows blueprint status, linked workspace, project tag, team wiring, joined artifacts, and whether an app still needs to be created.

  • Open this when a business idea becomes execution shape.
  • Project tags help Tasks, Workshop, and Agent Apps line up cleanly.
  • Use Apply Blueprint once a project is approved and ready to materialize.
10

Life Profile

A slow, document-first model of a person’s life context.

Life Profile separates recorded facts, missing-now gaps, and later context across the Life Improvement Team. It is built for incremental trust, not one giant intake.

  • Use it when advice needs better grounding than a chat window can hold.
  • Role-by-role dependencies stay visible instead of becoming implicit.
  • Best for careful support, not pressure.

Control and memory

Open these when you need org shape, access surfaces, editable files, or saved context.

11

Teams

The org chart view.

Teams turns saved configuration into managers, specialist lanes, lifecycle stages, incoming flow, support roles, and review loops that a human can actually read.

  • Use it when delegation feels messy and you need the structure back.
  • Live runs help show where a workflow is truly stuck.
  • Perfect for understanding how bundled teams fit together.
12

User Channels

Connect and govern real delivery surfaces.

User Channels is where quick setup, sender allowlists, group policies, mapped users, and provider-specific account access become manageable.

  • Use it after a channel is connected and before you widen access.
  • Good for WhatsApp, Telegram, Discord, and the other real message lanes.
  • Best when access rules need to match real relationships, not generic defaults.
13

Agents

Open the files and runtime scope behind an agent.

Agents lets you edit the durable voice and memory files, then flip into runtime scope when tools, file boundaries, or policies need inspection.

  • Use it when behavior feels off and you want the durable source, not a guess.
  • `SOUL.md` shapes voice and identity.
  • `MEMORY.md` holds persistent operational context.
14

Memory Notes

Filter recent saved memory by agent.

Memory Notes is the faster skim page. Instead of editing files, you can watch the latest saved notes for one agent at a time.

  • Use it when you want to understand what the system is actually retaining.
  • Helpful before editing memory so you do not duplicate what is already there.
  • Pairs naturally with Today when new notes just landed.
